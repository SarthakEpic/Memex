     1→     1→# Memex — Worklog
     2→     2→
     3→     3→## Project Overview
     4→     4→Building **Memex**, a citation-first knowledge retrieval system for personal Markdown notes, adapted from a FastAPI/Qdrant/Postgres/Ollama spec to our sandbox stack (Next.js 16 + Prisma/SQLite + z-ai-web-dev-sdk). The user also requested **email integration**.
     5→     5→
     6→     6→## Adaptation Decisions (sandbox constraints)
     7→     7→- **Backend**: Next.js API routes instead of FastAPI.
     8→     8→- **DB**: Prisma + SQLite instead of Postgres. BM25 done in JS (no `tsvector`).
     9→     9→- **Vector store**: No Qdrant / no embeddings API in z-ai SDK. Use **BM25 (TF-IDF) keyword retrieval** + optional **LLM rerank** for semantic matching. This keeps the citation-first contract intact.
    10→    10→- **LLM**: `z-ai-web-dev-sdk` `chat.completions.create` instead of Ollama/Llama.
    11→    11→- **Auth**: No Keycloak. Single-demo-user mode (current user derived from a fixed demo profile in DB).
    12→    12→- **Email**: Simulated SMTP — emails stored in DB `Email` table with status workflow (queued → sent → delivered), plus a digest scheduler. Real SMTP could be dropped in behind the same `sendEmail` interface.
    13→    13→- **Eval / Langfuse / RAGAS**: Out of scope for this sandbox build (no Python). Eval metrics surfaced in a lightweight in-app "Retrieval health" panel instead.
    14→    14→
    15→    15→## Differentiators preserved
    16→    16→1. **Citation-first generation** — every claim cites a chunk via `[^chunkId]`; unciteable claims become "I don't have a source for this."
    17→    17→2. **Decision timeline** — LLM extracts decisions from notes; queryable as "why did I decide X?"
    18→    18→3. **Honest eval** — surfaced as a retrieval-health dashboard (chunk count, avg terms, decision count, citation coverage) since RAGAS/Python isn't available here.
    19→    19→
    20→    20→## Email integration scope
    21→    21→- Compose modal reachable from Chat answers, Decision cards, and Notes ("Email this").
    22→    22→- Outbox + inbox view with status pipeline.
    23→    23→- Reusable templates (Daily Digest, Decision Brief, Source Snapshot).
    24→    24→- Scheduled digest job (every 15 min) that bundles recent decisions + unanswered questions into one email.
