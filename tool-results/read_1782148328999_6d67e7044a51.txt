     6‚Üí
     7‚Üí**Vision:** A retrieval-augmented chat system that answers questions from my personal Markdown notes and Git history, **cites the exact source chunk for every claim**, and exposes a **decision-timeline view** so I can trace why past technical choices were made.

     9‚Üí**Differentiators (only 3, do not add more):**
    10‚Üí1. **Citation-first generation** ‚Äî every claim in the chat answer is hyperlinked to its source chunk. If the model can't cite, it says "I don't have a source for this."
    11‚Üí2. **Decision timeline** ‚Äî LLM-extracted decisions queryable as "why did I decide X?" with source links.
    12‚Üí3. **Honest eval** ‚Äî 100-question hand-labeled golden set, Recall@K / MRR / RAGAS faithfulness reported on every PR.

   188‚Üí## PHASE 2 ‚Äî Markdown Ingestion + Naive Vector Retrieval (Week 2)
   189‚Üí

   191‚Üí
   192‚Üí### Database Schema (Alembic migration)
   193‚Üí
   194‚Üí```sql
   195‚Üí-- chunks: the atomic unit of retrieval
   196‚ÜíCREATE TABLE chunks (
   197‚Üí    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
   198‚Üí    doc_id UUID NOT NULL,                -- groups chunks from same file
   199‚Üí    source_path TEXT NOT NULL,           -- /notes/postgres-vs-mongo.md
   200‚Üí    source_type TEXT NOT NULL,           -- 'markdown' | 'pdf' | 'commit'
   201‚Üí    chunk_index INT NOT NULL,            -- 0,1,2,... within doc
   202‚Üí    text TEXT NOT NULL,
   203‚Üí    tsv TSVECTOR GENERATED ALWAYS AS (to_tsvector('english', text)) STORED,
   204‚Üí    metadata JSONB NOT NULL DEFAULT '{}', -- title, h1, h2, mtime, etc.
   205‚Üí    created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
   206‚Üí    UNIQUE (source_path, chunk_index)
   207‚Üí);
   208‚ÜíCREATE INDEX chunks_tsv_idx ON chunks USING GIN (tsv);
   209‚ÜíCREATE INDEX chunks_doc_id_idx ON chunks (doc_id);
   210‚Üí
   211‚Üí-- documents: per-file metadata
   212‚ÜíCREATE TABLE documents (
   213‚Üí    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
   214‚Üí    source_path TEXT UNIQUE NOT NULL,
   215‚Üí    source_type TEXT NOT NULL,
   216‚Üí    content_hash TEXT NOT NULL,          -- sha256, used for change detection
   217‚Üí    last_ingested_at TIMESTAMPTZ NOT NULL DEFAULT now(),
   218‚Üí    chunk_count INT NOT NULL DEFAULT 0
   219‚Üí);
   220‚Üí```

   222‚Üí### Services to Implement
   223‚Üí
   224‚Üí1. **`services/ingestion.py`** ‚Äî watches a configured directory (`/data/notes` mounted from host), detects new/changed Markdown files via content hash, enqueues RQ jobs.

   226‚Üí2. **`services/chunking.py`** ‚Äî Markdown-aware chunker:
   227‚Üí   - Split on H2 (`##`) boundaries first; if a section > 800 tokens, split by paragraph; if a paragraph > 800 tokens, split by sentence.
   228‚Üí   - Target chunk size: 512 tokens (use `tiktoken` for counting). Overlap: 64 tokens.
   229‚Üí   - Preserve heading path in metadata (`{h1: "Databases", h2: "Postgres vs Mongo"}`).
   230‚Üí
   231‚Üí3. **`services/embeddings.py`** ‚Äî async client for the BGE-M3 sidecar. Sidecar exposes `POST /embed` accepting `{texts: string[]}` returning `{embeddings: number[][]}` (1024-dim each).
   232‚Üí
   233‚Üí4. **`sidecars/embeddings/server.py`** ‚Äî FastAPI app loading BGE-M3 via `sentence-transformers`, exposing `/embed`. Run on CPU (or GPU if available). Batch up to 32 texts.

   235‚Üí5. **`services/retrieval.py`** ‚Äî naive version: embed query ‚Üí Qdrant search top 5 ‚Üí return chunks. No BM25, no rerank yet.

   274‚Üí
   275‚Üí   ```
   276‚Üí   You are Memex, a citation-first knowledge assistant. You answer questions

   280‚Üí   RULES:
   281‚Üí   1. Every factual statement MUST be followed by [^<chunk_id>].

   284‚Üí   3. Never use outside knowledge. Never speculate.

   286‚Üí   5. Keep answers under 200 words. Be concrete, not verbose.
   287‚Üí
   288‚Üí   CONTEXT CHUNKS:
   289‚Üí   [chunk_id: uuid-1]

   294‚Üí

   296‚Üí   source: /notes/infra-decisions.md
   297‚Üí   ---
   298‚Üí   Decided 2024-03-14: Postgres for primary DB. Mongo deferred.
   299‚Üí
   300‚Üí   QUESTION: Why did we pick postgres?
   301‚Üí
   302‚Üí   ANSWER (with [^chunk_id] citations):
   303‚Üí   ```
   304‚Üí
   305‚Üí3. **Post-processing citation verifier** (`services/llm.py:verify_citations`):
   306‚Üí   - Parse `[^chunk_id]` markers from the LLM output

   346‚Üí# Pseudocode ‚Äî implement for real
   347‚Üídef run_eval():
   348‚Üí    golden = [json.loads(l) for l in open('eval/golden_set.jsonl')]
   349‚Üí    results = []

   355‚Üí        # MRR ‚Äî reciprocal rank of first golden chunk

   358‚Üí            if cid in item['golden_chunk_ids']:
   359‚Üí                mrr = 1 / rank
   360‚Üí                break
   361‚Üí        results.append({'question': item['question'], 'recall': recall, 'mrr': mrr})
   362‚Üí
   363‚Üí    # Aggregate
   364‚Üí    avg_recall = sum(r['recall'] for r in results) / len(results)
   365‚Üí    avg_mrr = sum(r['mrr'] for r in results) / len(results)
   366‚Üí    p95_latency = percentile([r['latency_ms'] for r in results], 95)
   367‚Üí
   368‚Üí    print(f"Recall@5: {avg_recall:.3f}")
   369‚Üí    print(f"MRR:      {avg_mrr:.3f}")
   370‚Üí    print(f"p95:      {p95_latency}ms")
   371‚Üí
   372‚Üí    # Write to eval/results-{timestamp}.json for CI diff

   374‚Üí```

   396‚Üí## PHASE 5 ‚Äî BM25 + Reciprocal Rank Fusion (Week 5)
   397‚Üí
   398‚Üí**Goal:** Add BM25 keyword search via Postgres FTS, fuse with vector search via RRF. Recall@5 should jump from ~0.61 to ~0.74.
   399‚Üí
   400‚Üí### Implementation
   401‚Üí
   402‚Üí1. **`services/retrieval.py`** ‚Äî replace naive version with hybrid:
   403‚Üí
   404‚Üí   ```python
   405‚Üí   async def hybrid_search(query: str, top_k: int = 20) -> list[Chunk]:
   406‚Üí       # 1. BM25 (Postgres FTS)
   407‚Üí       bm25_results = await db.execute("""
   408‚Üí           SELECT id, ts_rank(tsv, plainto_tsquery('english', :q)) AS score
   409‚Üí           FROM chunks
   410‚Üí           WHERE tsv @@ plainto_tsquery('english', :q)
   411‚Üí           ORDER BY score DESC
   412‚Üí           LIMIT 50
   413‚Üí       """, {"q": query})
   414‚Üí       bm25_ranked = [r.id for r in bm25_results]  # ordered list
   415‚Üí
   416‚Üí       # 2. Vector (Qdrant)
   417‚Üí       query_emb = await embeddings_client.embed([query])
   418‚Üí       vec_results = await qdrant.search(
   419‚Üí           collection="memex_chunks",
   420‚Üí           query_vector=query_emb[0],
   421‚Üí           limit=50
   422‚Üí       )
   423‚Üí       vec_ranked = [r.id for r in vec_results]
   424‚Üí
   425‚Üí       # 3. Reciprocal Rank Fusion (k=60)
   426‚Üí       K = 60
   427‚Üí       fused_scores = {}
   428‚Üí       for rank, cid in enumerate(bm25_ranked, 1):
   429‚Üí           fused_scores[cid] = fused_scores.get(cid, 0) + 1 / (K + rank)
   430‚Üí       for rank, cid in enumerate(vec_ranked, 1):
   431‚Üí           fused_scores[cid] = fused_scores.get(cid, 0) + 1 / (K + rank)
   432‚Üí
   433‚Üí       # 4. Sort by fused score, take top 20
   434‚Üí       top_ids = sorted(fused_scores, key=fused_scores.get, reverse=True)[:top_k]
   435‚Üí       return await chunk_repo.get_many(top_ids)
   436‚Üí   ```
   437‚Üí
   438‚Üí2. **Re-run eval** ‚Äî commit before/after numbers to `EVAL.md`:
   439‚Üí   ```
   440‚Üí   | Metric          | Baseline | + BM25+RRF | Delta  |
   441‚Üí   |-----------------|----------|------------|--------|
   442‚Üí   | Recall@5        | 0.61     | 0.74       | +21%   |
   443‚Üí   | MRR             | 0.47     | 0.62       | +32%   |
   444‚Üí   ```

   446‚Üí### Acceptance Criteria
   447‚Üí

   765‚Üí## CROSS-CUTTING RULES (apply to every phase)
   766‚Üí
   767‚Üí### Code Quality
   768‚Üí- Every function has a type hint (Python) or TypeScript type (frontend). No `any` in TS, no untyped `def` in Python.
   769‚Üí- Every service-layer function has at least one unit test. Every API endpoint has at least one integration test.
   770‚Üí- `make test` runs all tests in < 60s. `make lint` (ruff for Python, eslint for TS) passes clean.
   771‚Üí- No `print()` in committed code ‚Äî use `structlog` for backend, `console.log` only in dev-only frontend code.
   772‚Üí
   773‚Üí### Database
   774‚Üí- Every schema change is an Alembic migration. No manual `ALTER TABLE`.
   775‚Üí- Every migration has a `downgrade()` that actually works (test it).
   776‚Üí- Foreign keys are explicit. Indexes on every column you `WHERE` or `ORDER BY` against.
   777‚Üí
   778‚Üí### Security
   779‚Üí- No secrets in code. `.env.example` has placeholders; `.env` is gitignored.
   780‚Üí- Keycloak tokens verified on every authenticated endpoint. No bypass for "internal" routes.
   781‚Üí- SQL injection: only SQLAlchemy parameterized queries. Never f-string SQL.
   782‚Üí- XSS: React escapes by default; never use `dangerouslySetInnerHTML` without sanitizing.

   787‚Üí- `/health` endpoint returns 200 + db/qdrant/redis/llm connectivity status.
   788‚Üí- Structured logs (JSON) for everything else.
